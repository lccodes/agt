CS100 Studio #0

Welcome!

Week of September 12th 2016

Instructions:

Welcome to your first CS100 studio! Today, we will be getting you ready for an exciting and productive semester. We will begin by signing course collaboration policies, and then we will set you all up with CS accounts (so that you can use the machines in the CS labs). Also on the agenda for today is getting you started with Markdown, a tool for creating web pages, and installing some software on your own personal laptops (if you happen to own one).

Upon completion of all of today’s tasks, a TA will give you credit for today’s studio. If you do not manage to complete all the assigned work during the studio period, do not worry. You can continue to work on this assignment until midnight on the Tuesday following this studio. Come by TA hours any time before then to show us your completed work and get credit for today’s studio.

Objectives

By the end of this studio, you will:

  • Sign your collaboration policy

  • Create a CS account

  • Join the course on Piazza

  • Learn about pair programming

  • Learn a little Markdown

  • Have fun finding some data

  • Discuss learning vs. inference with your partner

  • Download LibreOffice and install RStudio (Optional)

  1. Please see a TA to obtain a hard copy of the CS100 course collaboration policy.

  2. Read it, sign it, and return it to a TA. If you do not turn in a course collaboration policy, you will not get credit for today’s studio.

The Sunlab consultants will be on hand during today’s studio to set everyone up with a Computer Science Department account.

Piazza

Please sign up for our course on Piazza.

Later on in this studio, you will be posting on one of our course’s Piazza discussions. Additionally, throughout the semester, you can post questions on Piazza, and a TA will respond.

Pair programming

During future studio sessions, we will be “pair programming,” meaning that two students will be working together to complete the tasks. To get a better understanding of what to expect, please read this pair programming guide. Note that Amy originally compiled this guide for CS 17 / 18.

Once you have finished, please answer these True or False questions. You can jot your answers down on a piece of paper, or type them into a text file. Before you leave studio today, you will need to have them checked off by a TA.

  1. One student should be in charge of typing for the entire duration of a studio session.

  2. Pair programming is a divide-and-conquer strategy, in which each student works on a different part of the assignment.

  3. Pair programming helps students learn more and write better code.

  4. It is important to communicate respectfully while pair programming.

Markdown Tutorials

Later on in the semester (after we finish working with spreadsheets), we will require that your assignments be handed in in Markdown, a simple language for easily producing web pages.

Complete lessons 1-13 of this tutorial. (You need not complete lesson 14, since it teaches you how to insert code in Markdown, and you haven’t learned to write any code just yet.)

Fun finding data!

At some point soon, you will find yourself searching for your own data. So we’re going to practice now! At the end of Lecture 2a we demonstrated how to search for data specifying a topic and file type. Here’s a quick reminder:

You can use Google to search for CSV files by typing +filetype:csv in the search bar. For example, searching for South Africa +filetype:csv produces CSV files mentioning South Africa.

Find a CSV file on a topic you are interested in. Once you’ve found your data, please post it on Piazza. If you find something interesting, it might reappear later on in course material!

Learning vs. Inference

In class, we discussed two cornerstones of data analysis: learning and inference. Discuss with a partner how these concepts differ, and how they are the same. Then search (your own minds, or Google) for 1-2 examples of problems which might be better categorized as learning and then for 1-2 examples that might be better characterized as inference.

Please post your examples on the appropriate discussion on our Piazza page. (Amy might want to change the examples she presents next year, since as interesting as she thought they were, this year’s examples did not seem to generate much conversation.)
